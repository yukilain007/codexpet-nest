# CodexPet Nest v1.1 主题自动化验收指南

本文档介绍如何对遵循 Nest Theme v1.1 规范的主题进行结构化校验、安装测试以及渲染烟测。

## 验收目标

1.  **结构校验**: 确保 ZIP 包根目录包含 `codexpet-package.json` 和 `nest.json`。
2.  **Manifest 校验**: 验证 `codexpet-package.json` 中的 `type` 为 `nest`，且包含必要的元数据。
3.  **Layout 校验**: 验证 `nest.json` 符合 v1.1 schema，包括 Canvas 尺寸限制、元素定义和指标绑定。
4.  **安全校验**: 确保所有资产路径（`src`）均位于包内，无路径穿越风险。
5.  **渲染烟测**: 验证主题能否被 `LocalNestManager` 识别，并成功初始化 `NestRenderer` 及其子组件。

## 官方 Fixture 说明

项目在 `docs/test-fixtures/nests-v1.1/` 提供了三个官方示例主题，分别覆盖不同的 v1.1 能力：

-   **legend-status-nest**:
    -   覆盖 `MetricGaugeRenderer` (circleFill)。
    -   覆盖 `MetricTextRenderer` (百分比与单位)。
    -   覆盖多层背景层级。
-   **trainer-card-nest**:
    -   覆盖 `MetricGaugeRenderer` (linearBar)。
    -   覆盖 `MetricTextRenderer` (居中对齐、字号)。
    -   覆盖静态装饰图。
-   **window-desk-nest**:
    -   覆盖 `VariantImageRenderer` (基于昼夜状态切换资产)。
    -   覆盖 `widgetSlots` (在 v1.1 布局中嵌入内置组件)。

## 开发工具使用

### 1. 重新打包 Fixture

如果你修改了 `docs/test-fixtures/nests-v1.1/` 下的主题源码目录，需要重新生成 ZIP 包：

```bash
make package-nests-v1.1
```

该命令会自动剔除 `.DS_Store`、`__MACOSX` 等冗余文件。

### 2. 运行自动化验证

在开发环境下运行以下命令，将自动安装并测试上述三个主题：

```bash
make validate-nests-v1.1
```

**验证项包括**:
- ZIP 文件是否存在。
- `PackageManager` 安装逻辑是否通过。
- `LocalNestManager` 是否能刷新并识别。
- `NestRenderer` 在主线程初始化是否成功。
- `layout()` 调用是否正常。

### 3. 自定义路径验证

如果你想验证其他路径下的主题包，可以手动设置环境变量：

```bash
CODEXPET_VALIDATE_NESTS_V11=1 CODEXPET_NEST_FIXTURES_DIR=/your/path .build/CodexPetNest
```

## 已知限制

-   **视觉 QA**: 自动化工具只能确保“代码不崩”和“逻辑成立”。真实的视觉效果（如颜色是否好看、位置是否像素级精准）仍需开发者在 overlay 模式下人工检查。
-   **交互**: 自动化验证不包含点击测试或悬停反馈验证。
-   **性能**: 自动化验证为单次运行，不包含长时间运行的内存泄漏检查。
